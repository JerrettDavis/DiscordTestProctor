# Contributing

Thanks for taking the time to contribute to Janus Proctor.

## Getting started
1. Fork the repo and create a feature branch.
2. Follow the setup steps in `README.md`.
3. Keep changes scoped and include tests where practical.

## Development workflow
- `dotnet build` for the backend
- `dotnet test` for tests
- `npm start` in `src/Web/ClientApp` for the UI

## Coding standards
- Follow existing code style and formatting.
- Avoid adding secrets to source control. Use user secrets or environment variables.

## Pull requests
- Describe the problem, solution, and any tradeoffs.
- Include screenshots for UI changes.
- Note any migrations or configuration changes.
